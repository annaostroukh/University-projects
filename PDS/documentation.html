Documentation

Použité nástroje

Implementační jazyk aplikace tcpstats je Python verze 2.7.10 , webové stránky je HTML, CSS, JS.
Pro zpracování filu se zachycenými daty byla použita jednoduchá knihovna dpkt verze 1.8.7. Zejména třídy pcap.py - načtení PCAP filu; ethernet.py, ip.py, tcp.py - pro dolování dat.
Pro prezentace log souboru s analýzovanými daty ve formatu JSON slouží knihovna Highcharts.
Pro rychlejší vývoj a moderní vzhled webové stránky byl použit framework Bootstrap.

Použité zdroje

Jelikož knihovna dpkt nemá rozsáhlou dokumentaci, pro vývoj aplikace byly použity následující články:

  • Parsing a PCAP file
  • Jeff Silverman dpkt documentation
  • Data Mining PCAP files

Informace o TCP spojení

Pro analýzu byl použit PCAP file z referenčního článku TCP Slow Start k zadání projektu. Jak je zmíněno ve článku, pcap obsahuje data zachycená v průběhu stážení filu velikosti 1 MB přes protokol HTTP ze serveru.

Sumarní statistiky

File slow-start.pcap obsahuje 1246 paketů a celkový čas spojení je 0.77 sekund. Spojení probíhalo mezi klientem s IP adresou 192.168.1.40 a serverem s IP adresou 174.143.213.184.

Výsledné statistiky

Pro analýzu statistik, výsledky kterou jsou reprezentovaný na grafech, byla použita následující získaná z pcap filu informace:

  • Window
    • Velikost window bufferu klienta/serveru
    • Scale faktor získaný na začátku spojení (flagy SYN a SYN ACK)
    • Časový okamžik přenosu
    • Flagy paketů odlišné od SYN a SYN ACK
  • Rychlost spojení
    • Celkový počet přenesených bitů za zvolený časový interval (0.01 s)
  • Round trip time
    • Přítomnost u paketů vhodného flagu ACK
    • Směr spojení od klienta k serveru
    • Hodnota sekvenčního čísla odpovídající hodnotě acknowledgmentu
  • Analýza sekvenčních čísel
    • Přítomnost u paketů vhodných flagů SYN ACK/SYN, ACK, FIN ACK/FIN
    • Hodnota sekvenčního čísla (jsou použity absolutní hodnoty z pcap filu)
    • Časový přírůstek od SYN ACK paketu
